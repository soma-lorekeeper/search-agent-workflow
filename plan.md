# 소설 질의응답 에이전트 PoC — 개발 계획

## 목표


`agentic-workflow/data/episode1~6.txt` (『전지적 독자 시점』 실제 원문)를 인덱싱하고, 아래 "정답지"의
7개 target query 전부에 대해 사실에 부합하고 근거 없는 내용(할루시네이션)이 없는 응답을
생성하는 LangGraph 기반 질의응답 에이전트를 만든다. 7개 전부를 만족할 때까지 반복 개발한다.

이 작업은 `mvp/`(Neo4j 기반 설정 충돌 탐지 파이프라인)와 독립적이며, 어제 작성된
`docs/superpowers/specs/2026-07-10-vectorrag-agentic-loop-design.md`(단순 ReAct+Qdrant 프로토타입)
도 참고만 하고 직접 재사용하지 않는다 (이번엔 구조화 저장소가 추가되고, 검색 판단 로직이
더 들어가기 때문).

## Target Query & 정답지 (Ground Truth)

에이전트 응답이 "정답지의 핵심 사실을 포함하는가" + "정답지에 없는 사실을 지어내지 않는가"
두 기준으로 채점한다.

### Q1. (Vector / 서사 회상) 
"1화에서 김독자가 첫 시나리오를 돌파한 방법이 뭐였는지, 그리고 비슷한 방식(회유/우회)을
이후 화에서도 다시 쓴 적이 있어?"

**정답**: 1화에서 김독자는 사람을 직접 죽이지 않고 곤충과 메뚜기 알을 이용해 '가치 증명'
시나리오의 살인 조건을 우회해 코인을 얻었다. 4화에서 철두파(방철수 등)와 싸울 때도
"클리어 조건은 어디까지나 무력화"라며 직접 죽이지 않고 제압하는 동일한 패턴을 반복했다.

### Q2. (Graph 다중홉 / 아이템 이력)
"'부러진 신념'이라는 아이템 어떻게 손에 넣었고, 최종적으로 어떻게 됐어?"

**정답**: 3화에서 어룡을 잡은 뒤 어룡의 핵을 도깨비 보따리 거래소의 교환 경매에 등록했다.
5화에서 그 대가로 D등급의 부러진 칼 '부러진 신념'을 받았고, 별도로 구매한 스킬
'백청강기'(1만 코인)를 주입해야 사용 가능했으며 어둠파수꾼과의 전투에 썼다. 6화에서
한정판 랜덤 아이템 박스에 '부러진 신념'을 제물로 넣어, 성유물급 아이템 '부러지지 않는
신념'(카이제닉스의 검, 체력·근력 +2)으로 업그레이드되었다. 최종 소유자는 김독자.

### Q3. (캐릭터-소유 이력)
"한명오가 유상아의 물건을 몰래 가져간 적이 있어?"

**정답**: 있다. 5화에서 다툼 중 유상아가 "제 자전거 부장님이 훔쳐 가셨냐고요"라고 추궁하자
한명오가 시인했다("그래 씨발! 내가 그랬다!"). 1~4화에는 언급되지 않고 5화에서 처음 드러난다.

### Q4. (상태 변화 / 설정 충돌 탐지형 — 정보 부재를 정확히 인지해야 하는 케이스)
"이현성의 능력치(체력/근력/민첩/마력)가 화가 지날수록 어떻게 바뀌었어?"

**정답**: 4화에서 처음 공개된 수치는 체력12·근력9·민첩9·마력6(스타터팩 적용 상태)이다.
6화에서는 개별 레벨이 다시 공개되지 않고 "체근민 합 37"이라고만 언급된다. 즉 개별 능력치의
정확한 변화값은 원문에 없다 — 에이전트가 "6화 시점 개별 수치는 원문에 없고, 합계 37만
확인된다"처럼 정보 부재를 인정해야 정답이며, 임의로 개별 수치를 추정해 단정하면 오답.

### Q5. (인물 관계 / 사건)
"천인호는 몇 화에서 죽어? 누구 손에?"

**정답**: 6화. 정희원이 특성을 '멸악의 심판자'로 개화한 직후, 철두파를 학살하는 과정에서
천인호를 살해한다.

### Q6. (멀티턴 지시어 해소)
1턴: "정희원이라는 캐릭터 소개해줘"
2턴: "이 캐릭터가 예전에 어떤 물건을 남한테 나눠준 적 있어?"

**정답**: (1턴) 정희원은 4화에서 김독자가 맹독 안개 속에서 구조한 인물로, 검도 경험이 있고
특성 '웅크린 자'가 6화에서 '멸악의 심판자'(영웅급)로 개화, 전용 스킬 '심판의 시간'을 가짐.
(2턴) 6화에서 철두파에게서 빼앗은 코인 중 1500코인을 김독자에게 나눠줬다("혼자 먹기
미안해서, 다른 사람들한테도 줬어요"). 2턴에서 "이 캐릭터"가 정희원으로 정확히 바인딩되는지가
핵심 검증 포인트.

### Q7. (부정 케이스 — 할루시네이션 방지 함정 질문)
"김독자가 유중혁과 정식으로 계약을 맺은 적이 있어?"

**정답**: 없음. 2화에서 유중혁이 동료 제안(배후 선택)을 했지만 김독자는 마지막 시험에서
결국 어룡에게 던져졌고, 정식 계약이 성사된 적은 원문에 없다. 검색 시 2화의 유중혁 관련
장면이 관련성 높게 걸리더라도, "계약했다"고 지어내면 오답.

## 아키텍처

### 저장소

- **VectorDB (Qdrant)**: 화 원문을 청크 단위로 임베딩해 저장. 의미 기반 회상(Q1, Q6-1턴)에 사용.
- **구조화 저장소 (MySQL)**: Neo4j/Cypher 대신 관계형 테이블로 그래프 성격 데이터를 표현.
  최소 스키마:
  - `entities(id, name, type, episode_introduced)` — 인물/아이템
  - `facts(id, subject_id, predicate, object_id_or_text, episode, valid_from_episode, valid_until_episode, note)`
    — 소유/관계/사건을 주어-관계-목적어 형태로 저장, 화 번호를 시간축으로 사용
  - `episode_summaries(episode, summary)` — mvp의 롤링 서머리 방식 재사용 (있으면 그대로 활용)

  Q2(아이템 이력), Q3(소유 이력), Q4(능력치), Q5(사건), Q7(관계 부재 확인)은 이 테이블에
  대한 SQL 조회로 답한다. 다중홉(Q2)은 동일 entity_id를 여러 `facts` row에서 추적하는
  JOIN/재귀 조회로 해결 — Neo4j 없이도 이 정도 깊이(2~3홉)는 충분히 처리 가능하다는 전제.

- Qdrant, MySQL 모두 `docker-compose.yml`로 로컬 기동. 인증은 mvp와 동일하게 로컬 개발용
  최소 설정.

### 인덱싱

기존 mvp 인덱서(Cypher 생성용)는 재사용하지 않고, 이번 스코프에 맞는 새 인덱서를 만든다:
원문 → LLM이 (a) 챕터 청크 임베딩 대상 텍스트, (b) entities/facts용 구조化 추출(JSON) 을
동시에 생성 → (a)는 Qdrant upsert, (b)는 MySQL insert.

### 에이전트 루프 (LangGraph)

- `agent` 노드: 질문 판단 + 필요한 tool 호출 결정 (ReAct 기반)
- Tools: `vector_search(query, top_k)`, `structured_query(question)` — 후자는 LLM이 자연어
  질문을 SQL로 변환해 실행하는 Text2SQL 방식 (Text2Cypher의 SQL 버전)
- `agent ↔ tools` 루프에 **grade 단계**를 추가: 검색 결과가 질문에 답하기 충분한지 판단,
  불충분하면 쿼리를 바꿔 재검색 (Self-RAG/CRAG 아이디어 반영 — 브레인스토밍 단계에서
  논의한 내용)
- 멀티턴(Q6)을 위해 LangGraph checkpointer로 대화 상태 유지 + 질문 재작성(query rewrite)
  노드로 "이 캐릭터" 같은 지시어를 이전 턴에서 언급된 엔티티로 치환

## 개발 방식 (에이전틱 개발 — 중구난방 방지)

1. 저장소(Qdrant+MySQL) 스키마 확정 후 docker-compose로 기동
2. 인덱싱 파이프라인 구현 → 6개 화 인덱싱 실행 → 결과 수동 스팟체크(주요 엔티티/관계 누락
   없는지)
3. 개별 tool(vector_search, structured_query) 단위로 먼저 검증 — 각 tool을 target query에
   필요한 최소 조회로 직접 호출해보고 정답지와 대조
4. LangGraph 에이전트 루프 조립
5. **평가 루프**: 7개 query를 순서대로 실행 → 응답을 정답지와 대조 (핵심 사실 포함 여부 +
   허위 사실 여부) → 실패하면 원인 분류(검색 실패/tool 선택 실패/스키마 부족/프롬프트 문제)
   → 해당 계층만 수정 → 재실행
6. 7개 전부 통과 시 완료. 완료 조건: 모든 query에 대해 정답지 핵심 사실 포함 + 허위 사실 없음.
